# 產品需求文件 (PRD) - 校園美食推薦系統

## 1. 專案概述

- **背景與動機**：學生每天經常為了「下一餐要吃什麼」而煩惱，學校周圍雖然有許多餐廳，但資訊多半散落或僅依賴同學間的口耳相傳，缺乏一個可以統整附近餐廳資訊、查看真實評價的專屬平台。
- **目標用戶**：高中生與大學生。
- **核心價值主張**：提供一個學生專屬、真實可信的校園周圍美食資訊平台，幫助學生快速找到符合預算、距離與口味的餐廳。

## 2. 功能需求

以下列出系統的主要功能及對應的使用者故事 (User Story)：

1. **使用者註冊與登入**
   - 作為一名學生，我希望能夠註冊與登入帳號，以便在平台上發表評論或是收藏喜愛的餐廳。
2. **餐廳瀏覽與分類**
   - 作為一名學生，我希望能夠依據價格範圍、距離遠近以及食物類型來分類與篩選餐廳，以便快速找到適合當天需求的選擇。
3. **餐廳詳細資訊**
   - 作為一名學生，我希望能夠點擊進入餐廳頁面，查看詳細的菜單、照片及營業時間，以便決定是否前往消費。
4. **評分與評論系統**
   - 作為一名學生，我希望能夠對吃過的餐廳留下真實星星評分和文字評論，以便幫助其他同學推薦美食或避開雷店。
5. **收藏餐廳功能**
   - 作為一名學生，我希望能夠將喜歡或想去吃的餐廳加入個人的收藏名單中，以便未來不知道吃什麼時可以馬上查看。
6. **關鍵字搜尋**
   - 作為一名學生，我希望能夠透過輸入關鍵字搜尋特定的餐廳名稱或食物關鍵字，以便精準、快速地找到我想找的店家。

## 3. 非功能需求

- **技術限制**：
  - 後端框架：Flask (Python)
  - 前端渲染：Jinja2 模板引擎
  - 資料庫：SQLite (輕量級關聯式資料庫)
- **效能與安全考量**：
  - **安全性**：使用者密碼必須經過雜湊 (Hash) 加密儲存；表單輸入必須防範常見的 SQL Injection 及 XSS 攻擊。
  - **效能**：系統必須能在低資源環境下流暢運行，確保學生透過手機或電腦瀏覽餐廳、搜尋分類時能有快速的響應速度。

## 4. MVP 範圍 (Minimum Viable Product)

- **Must Have (必須有：系統核心)**
  - 餐廳詳細資訊 (含店名、介紹、營業時間等)
  - 餐廳瀏覽與所有餐廳列表展示
- **Should Have (應該有：提升實用性)**
  - 使用者註冊與登入 (能辨識發布來源)
  - 評分與評論系統 (產生 UGC 內容)
  - 餐廳分類與過濾 (價格、距離、類型)
- **Nice to Have (加分項目：優化體驗)**
  - 關鍵字搜尋
  - 收藏餐廳功能
